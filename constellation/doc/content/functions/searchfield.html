SearchField plugin

  • Required files:
    common.js, jquery.accessibleList.js, searchfield.js
  • Available in:
    standard + mobile

About searchField plugin

This plugin is used to extend the template search field, offering real-time search and graphical results display. As soon as the user starts typing, requests are sent to the server and results are displayed in real-time. The plugin can also search within the template main navigation, thus providing immediate results (this behaviour cna be disabled).

It relies on the accessibleList plugin to collapse too long results, so the list always fit on the screen.

To use it, just call:

$(selector).advancedSearchField(options);

options is an object with any of the options below.

The plugin is called automatically at startup in the file standard.js.

Available options

minSearchLength
Minimum search string length (default: 2)
moreButtonAfter
Max number of visible matches in each list above 'more' button (0 for no masking) (default: 3)
matchesPerPage
Number of visible matches per page (0 for no pagination) (default: 5)
firstRequestDelay
Delay before first request in ms (avoid multiple request while user types) (default: 250)
nextRequestDelay
Minimum delay between search requests in ms (reduces server load) (default: 750)
noUpdateDelay
Minimum delay upon which the plugin considers the user has stopped typing long enough to start the next request with firstRequestDelay delay, thus responding faster (default: 3000)
enableNavSearch
Enable search within template navigation (default: true)
tipFocus
Tip when user focuses the search field (default: 'Enter your search')
tipTooShort
Tip if search is too short (default: 'Enter at least 2 chars')
messageLoading
Message while loading (default: 'Loading results...')
messageNoMatches
Message when no matches found (default: 'No matches')
messageSearchDone
Message when the search request is done (default: 'Not found? <a href="#">Try advanced search »</a>')
messageErrorFull
Full message if an error occurs while loading results (default: 'Error while loading results, please try again')
messageError
Status message if an error occurs while loading results (default: 'Error while loading')
titleTemplateResult
Title of the result section showing template nav items (default: 'Admin pages')

You can change the function's defaults at runtime like this:

// Example: change the minimum search length
$.fn.advancedSearchField.defaults.minSearchLength = 4;

Search script

The plugin will automatically call the page defined by the form's action attribute, or the page itself if the attribute is empty.

To create your own search script, retrieve the user search string in the $_POST['s'] var. You can find an example of formatted results in the file search.html/search.php.

Search list markup

Here is an example of result list markup:

<!-- The main block - created by the plugin -->
<div id="search-result" class="result-block">
	<!-- Top arrow -->
	<span class="arrow"><span></span></span>
	
	<h2>Admin pages</h2>
	<p class="results-count"><strong>7</strong> matches</p>
	<ul class="small-files-list icon-html">
		<li>
			<a href="#"><strong>Add</strong> new user<br>
			<small>Users > Add new user</small></a>
		</li>
		<li>
			<a href="#"><strong>Add</strong> new image<br>
			<small>Write > Library > Add new image</small></a>
		</li>
		<li>
			<a href="#">Recently <strong>add</strong>ed comments<br>
			<small>Comments > Recently added comments</small></a>
		</li>
	</ul>
	
	<!-- This element is created by the plugin AccessibleList -->
	<a href="#" class="search-more">More</a>
	<hr>
	
	<h2>Website pages</h2>
	<p class="results-count"><strong>1</strong> match</p>
	<ul class="small-files-list icon-xml">
		<li>
			<a href="#">How to create an <strong>add</strong>-on<br>
			<small><strong>Author:</strong> John Doe</small></a>
		</li>
	</ul>
	
	<hr>
	
	<h2>Images</h2>
	<p class="results-count"><strong>1</strong> match</p>
	<ul class="small-files-list icon-img">
		<li>
			<a href="#">Create_<strong>add</strong>-on_illustration_1.jpg<br>
			<small><strong>Size:</strong> 320x240</small></a>
		</li>
	</ul>
	
	<!-- Status line -->
	<p id="search-info" class="result-info loading">Not found? <a href="#">Try advanced search »</a></p>
</div>
