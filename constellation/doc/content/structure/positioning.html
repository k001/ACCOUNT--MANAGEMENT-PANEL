Positioning

Hierarchy

The template uses many positioned elements, such as menus, overlapping divs and so on. To avoid z-index issues, it uses a strict z-index hierachy:

88 or 89
Positioned elements at normal level
98 or 99
Positioned for menu
100
Footer (standard template only)
999900
Tooltip
999910
Menu
999950
Fixed control bar
999990
Notifications

Elements positioning

All template positioned styles already have their z-index set up according to this hierarchy. However, if you need to position a custom element, you may use this two classes:

.absolute
Class with position:absolute and z-index:89
.relative
Class with position:relative and z-index:89

In some case (mostly drop-down menus), you'll need an element to have a slightly higher index as the following ones, so menus (for instance) won't get covered. In these cases, add the class upper-index to the element to give it a z-index of 100.

This way, you don't have to remind which z-index to use.
