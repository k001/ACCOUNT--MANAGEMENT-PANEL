Special pages

About special pages

The template includes several special pages:

login
User login page with ajax demo script (with server-side script on the php version)
wizard
Example page for a setup wizard
old-browsers
Blocking page displayed automatically to older browsers to prevent them from accessing the application
_errors/error
Example error page, with detailed error report. The php version is also fully fonctional - see file error.php at the root of the Php template for usage example - and includes report sending through an ajax request to file sendReport.php (see below).
_errors/sendReport
Example page for sending error report. The php version works, in association with error.php - just add mail sending of logging depending on your needs.
_errors/xxx

Error documents, can be used for instance with Apache webserver with an .htaccess file (provided in the php version) or any other HTTP server:

400: Bad request
401: Unauthorized
403: Forbidden
404: Page not found
500: Internal server error
503: Service Unavailable

All pages (except the one for the older browsers) are compatible with both standard an mobile template, by using CSS media Queries.

These pages (except the one for the older browsers) use the template standard classes, and a custom CSS file. Just note that they also have the class special-page applied to the <body> element: this is only required for script optimizations.

Also note that all pages in folder _errors/ in the php version (except sendReport.php) are intended to be loaded either by the HTTP server or by a php script, so relative pathes to CSS/JS/images won't work if you open them directly.

Error documents

To use the template error documents with an Apache HTTP server, make sure that you can use .htaccess files, and create a file named .htaccess at the root of your website (or edit it if it already exists):

ErrorDocument 400 /_errors/400.php
ErrorDocument 401 /_errors/401.php
ErrorDocument 403 /_errors/403.php
ErrorDocument 404 /_errors/404.php
ErrorDocument 500 /_errors/500.php
ErrorDocument 503 /_errors/503.php

Note that if your website is in a folder under the root of your hosting, you'll need to provide the absolute path (relative pathes won't work):

# Example with folder /admin/
ErrorDocument 400 /admin/_errors/400.php
ErrorDocument 401 /admin/_errors/401.php
ErrorDocument 403 /admin/_errors/403.php
ErrorDocument 404 /admin/_errors/404.php
ErrorDocument 500 /admin/_errors/500.php
ErrorDocument 503 /admin/_errors/503.php
