Template versions

CSS/JS structure

The template comes in 2 versions:

Standard
This is the default template, for computer screens.
Mobile
This template is designed for webkit based mobile browsers (like iPhone, Android...), and is compatible with finger interactions: large buttons, special menu...

Both templates share a number of CSS styles and jQuery functions: they are located in the files common.css and common.js.

The version-specific styles (menus, bars...) are gathered in the files standard.css and mobile.css, as for the version-specific jQuery functions in the files standard.js and mobile.js. Note that there is also a file standard.ie.js, needed to handle missing CSS selectors for Internet Explorer 7 & 8.

About Internet Explorer

To allow easy handling of IE-specific styles without using css hacks, the template uses 2 nested conditionnal divs:

<body>
<!-- The template uses conditional comments to add wrappers div for ie8 and ie7 - just add .ie or .ie7 prefix to your css selectors when needed -->
<!--[if lt IE 9]><div class="ie"><![endif]-->
<!--[if lt IE 8]><div class="ie7"><![endif]-->

	[Content]

<!--[if lt IE 8]></div><![endif]-->
<!--[if IE]></div><![endif]-->
</body>

How does it work?

With conditionnal comments, only Internet Explorer will see these divs: IE7 and below will only see the <div class="ie7">, IE8 will see both.

Targetting a specific version of IE can be done just by adding a small selector:

.mySelector {
	/* This will apply to all browsers */
	}
	
	.ie7 .mySelector {
		/* This will apply to IE7 and below */
	}
	
	.ie .mySelector {
		/* This will apply to IE8 and below */
	}

The jQuery script in standard.ie.js will also automatically add classes first-child and first-child to every first and last childs, replacing the missing :first-child and :last-child selectors. Note that because IE doesn't understand them at all, you'll have to split the style selectors, otherwise it won't work:

/* This won't work in IE */
.mySelector:first-child,
.mySelector.first-child {
	margin-top: 0;
}

/* This will work */
.mySelector:first-child {
	margin-top: 0;
}
.mySelector.first-child { /* IE class */
	margin-top: 0;
}

Note that IE9 (currently in alpha version) does not require such targetting, because it handles all required CSS3 selectors
