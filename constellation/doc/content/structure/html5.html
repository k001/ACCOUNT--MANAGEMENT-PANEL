About HTML5

HTML5 is the next big thing in webdesign. Although it's final version (WC recommendation) is not to be expected before 2022, all modern browsers already support it, and using it grants your web application a longer lifetime.

Backward compatibility

This template uses a few HTML5 features (mainly new HTML nodes types), and to achieve compatibility with older browsers, it uses these 2 hacks:

CSS declaration
Located in the file reset.css, this hack simply consists on declaring a default behaviour for the new HTML5 nodes, so older browsers will display them properly
/*
 * Default HTML5 behaviour for older browsers
 */
article, aside, audio, canvas, command, datalist, 
details, embed, figcaption, figure, footer, header, 
hgroup, keygen, meter, nav, output, progress, 
section, source, video { display: block; }
mark, rp, rt, ruby, summary, time{ display: inline }
JS declaration
The file html5.js forces the browser to parse new nodes correctly, by creating an instance of each. Note that IE wouldn't behave otherwise.
document.createElement("article");
document.createElement("aside");
[...]
document.createElement("time");
document.createElement("video");

What does it changes?

Besides new node types, HTML5 brings a lot of new features: local storage, geolocation, drag & drop... - you can go here for some demos: html5demos.com/.

Regarding this template, the only thing you need to know about HTML5 is that the markup is slightly different from previous HTML versions. Here is an excerpt of the changes you'll see in the sourcecode:

Simplified doctype
No need for complicated declarations anymore: just declare your document as HTML and it's all!
<!DOCTYPE html>
<html lang="en">
[...]
</html>
New charset meta
Declaring the document charset has never been easyier:
<meta charset="utf-8">
No trailing slash for empty nodes
<br>
<img src="image.png">

Some HTML5 nodes used here...

<article>
This node is used as the main content wrapper
<section>
Defines a content block
<header>, <nav>, <footer>
Used as logical structural elements
<progress>, <meter>
Used to display progress bars. Not that <meter> is to display a current state/level, <progress> to show progression
